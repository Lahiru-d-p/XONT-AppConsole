Route Transaction

 

Invoicing or order taking functionalities in Ventura web application could be attained through this component. Initial screen, which requires the executive to login to the route transaction, will be displayed as below.

1st Screen

  In order to perform rout transaction, executive needs to login to the component by entering the executive code as the user name and the password. After entering relevant details, user can tap on Login button in order to start the transactions.

 

  If the executive is a POS executive, user will be redirected to the Retailer list. Otherwise displays the list of routes.

 

  Below image shows the retailers list where the POS user will be redirected.

  Routes list will be displayed as below, which is loaded to an order taking type of user.

 

  Routes which have been planned on the given date will be highlighted in green colour, and status will be displayed as default route as shown in the above figure (Routes can be attached to an executive by adding the executive as the default route executive in Route Maintenance. Similarly, route plans can be done through Route Itinerary option).

 

  User can select Show miscellaneous Routes option in order to view/select the routes of other executives in the same territory.

Please Note that this option will be enabled only when Miscellaneous Retailer option is enabled in Territory Control (Refer: Territory Control Maintenance).

 

  After selecting a Route, user should click on the OK button in order to navigate to the next screen. When the user selects the route for the first time for a particular date, following window will appear.

  Details such as Start time and date will be filled with the system date and time, where the user can enter mileage details, vehicle number, driver, porter details.

 

  After entering relevant details, user should click on OK in order to move to the following screen.

 

  Retailers could be filtered according to the selection criteria.

 

  Clicking on Complete will allow the user to complete the sales and move to the previous interface which allows selecting the routes.

 

  No Sale option can be selected if there is no transaction at the dealer point. No Sale option will pop up the following window which requires the user to select the call lost reason through the prompt and click OK to move to the route listing interface.

 

  Selecting the retailer and then clicking on the Invoice option will check factors such as whether customer blocking criteria are available retailer on stop flag is enabled.

 

 

  The product load screen will include details such as Retailer Code, warehouse and location codes, executive, sales category, Settlement Term, Invoice date, retailers credit limit, outstanding and available credit.

 

  Products could be filtered using the selection criteria. Selections could be based on product classifications, product code and product description.

 

 

Please Note that,

If user maintains POS Date Change Flag true in Control maintain system will allow changing the Invoice Date.

According to the Product Load Type in Territory Control system will load all products, retailer products, executive products or retailer executive product when user clicks List button.

System will not validate the order stock as long as the order stock validation flag is false and the system is order taking (Refer: Territory Control Maintenance to set order stock validation flag).

Line, Header and Special discounts will be applied, only if Line Discount Active, Header Discount Active, and Special Discount Active flags are enabled in Territory Control accordingly (Refer: Territory Control Maintenance).

 

  User can enter the order/Invoice quantity in the field against each product and move to the next page or change discount or batch.

 

Editing Discount and Price

 

Clicking on this option will display the following screen which allows the user to edit price and discount of the products.

 

  User can select a line and click on Price Override button to change the prices, which will be redirected to following page.

  User can enter the New Price and click on the Update button in order to update the price. Should click on OK to confirm the price change.

Please note that price changes could be done only if POS Price Change flag is active in RD Control (Refer: RD Control Maintenance). If POS Price Change flag is set as Not Allowed, then the user wont be able to change the price.

 

  Similarly, Line Discounts can be changed by selecting a record and clicking on Disc.Override button, which will display the following screen.

 

  User can enter the Discount Amount, Product and Discount Reason code and click on the Update button.

 

  If user wants to enter more than one line of discounts, user can click on Add new line button.

Add New Line option will be enabled only if Add Line Lvl Free Issue with Reason Code flag is enabled in territory control.

Please note that the user will be able to do discount override only according to Line Discount Overwrite flag in Territory Control Maintenance (Refer: Territory Control maintenance). If line discount overwrite flag is set as Not Allowed, then the user wont be able to overwrite line discount.

  After entering relevant details, user can click on Ok button to navigate to previous screen.

 

Editing Header Discount

 

  User will be able to overwrite header discount by clicking on the Edit Header Discount option. Clicking on this option will display following screen where the user will be able to enter header discount attributes.

  If header discount exists, it will be displayed and the user can click Edit Discount to edit the existing discount or Add/Edit special discount to add special discount.

Please note that user will be able to overwrite header discount only according to the Header Discount Overwrite flag in Territory Control (Refer: Territory Control maintenance). If Header discount overwrite flag is set as Not Allowed, then the user wont be able to overwrite the header discount.

Editing Batch

 

User can edit the batches of the product by clicking on the Edit Batch option. When user selects the Edit Batch button, user will be redirected to another screen where the order/ invoice products will be listed.

When user selects the product, batch popup will be prompt where the user can change the batch quantities, as shown below.

After entering batch quantities, user should click OK to save the changes.

 

v  After entering relevant details, user can move to the next main interface by clicking on Next button.

Clicking on Next button will validate the inputs and redirect the user to another page which displays the Order/Invoice Details.

  Details such as Order/ Invoice products, quantity, discount, free issue details and Tax details will be displayed here.

Please note that tax will be applied only if VAT Apply flag is enabled in RD Territory Control (Refer: RD Territory Control maintenance).

 

  User can also enter details such as Transport reference, Extra charge values, extra charge Details, Payment method and Remarks.

  Finally, User can click on Save button to save the order/ invoice or Back button to move to the previous screen.

  Make sure the prices are defined for products.

  Make sure to set relevant flags which affect the transaction correctly in territory control and RD control.

  Set proper executive types, whether they are order taking or invoicing.
