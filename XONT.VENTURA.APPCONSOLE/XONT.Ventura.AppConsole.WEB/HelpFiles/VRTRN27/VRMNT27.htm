Distributor Maintenance

 

This Component is used to maintain details related to Distributors in each territory.

At the initial load screen all available Distributors will be listed in the grid.

1st Screen

v  By expanding the Selection Criteria, User will be able to filter Distributors accordingly.

Selection could be based on Distributor Code, Territory and other distributer classifications. Results could be filtered using Start with or Any where in Text option. User can also view the active and inactive records by unticking the Active check box.

 

v  There are different Options associated with the Main screen.

 

Add New Distributor to the System

  Click New Button

 

Add New Distributor Based on an existing Distributor

  Select specific Distributor New Based On Button

 

Edit specific Distributor in the System

  Select Distributor Edit Button

 

Exit from Distributor Maintenance

  Click Exit Button

 

v  In the Distributor Creation and Editing Process following fields are mandatory.

  Distributor Code

  Distributor Name

  Address

  Zip/Post Code

  Alpha Search Code

  Short Address

  VAT Code

  Price Group

  Trade Scheme Group

  Pricing Method

  Territory Code

  Settlement Terms Code

  Payment Method Code

Creating a new Distributor

Creating A completely New Distributor

 

  User can create a completely new Distributor by clicking the New button.

 

 

  When clicking the new button, following interface appears and the user will be able to enter the Distributor attributes.

 

 

 

 

 

 

 

2nd Screen

 

  Distributor Code: A unique code which allows the user to identify the Distributor and differentiate from other Distributors.

 

  Distributor Name: Allows the user to enter the Distributor name.

 

  Address Line 1/2/3/4/5: User should enter the address of the Distributor in this field. It is mandatory to add at least one address line to the system.

 

  Zip/Post Code: This is a mandatory field which requires adding the Post Code of the Distributor.

 

  Alpha Search Code: Allows entering the Alpha search code of the Distributor.

 

  Contact Name: Allows adding the Contact Name of the supplier if exists.

 

  Short Address: The user should enter the short address of the Distributor in this field.

 

  Fax Number: Allows adding the fax number of the supplier if exists.

 

  Telephone Number: Allows to add the telephone number of the supplier if exists.

 

  Email Address: Allows adding the email Address of the supplier if exists.

 

  Web Address: Allows adding the web address of the supplier if exists.

 

  VAT Code: This is a mandatory field which requires the user to enter the VAT code through the prompt. These VAT codes will be maintained in Tax Details. (Refer Tax Details)

 

  Registered: This checkbox determines whether the distributor is VAT registered.

 

  VAT Registration Number: This field will be enabled only if Distributor if VAT registered.

 

  Active: This checkbox determines whether the Supplier is active or inactive in the system. This checkbox will be automatically checked when creating a new supplier. User can also select whether the Distributor is Normal, Black Listed, On stop or On Trial.

 

 

 

 

 

 

 

 

 

v  Classification tab

  Distributor Classifications: User will be able to select the Distributer Classifications such as Type, Class and Category, from the predefined classification values. (Refer: Classification Code, Classification Value)

  Price Group: This is a mandatory field which requires selecting the Price Group of the distributer, using the prompt.

  Trade Scheme Group: Allows the user to select a Trade Scheme group from the prompt.

  Pricing Method: This field requires the user to select the pricing method.

  Default Sales Rep: Allows the user to select the default sales rep, if exists.

  Commission Percent: Allows the user to add commission percent if exists.

  Commission Act Code: Select the Commission Act code if exists.

  Territory Code: Mandatory field which requires the user to select the territory code of the distributor.

  Default Supplier: Allows to select the default supplier if exists.

  User can also add the details such as Source Warehouse, Source W/H Location, Destination Warehouse and Destination W/H Location if exists.

 

 

v  Other Details tab

  This tab allows the user to enter details such as Country Code, Mode of Transport, Terms of Delivery, Invoice Declaration, Consolidated Invoice Code and Invoice Account Code.

  Settlement Terms Code: Mandatory field which requires the user to determine the settlement terms code of the Distributor.

  Payment Method Code: Mandatory field which requires the user to determine the Payment Method code of the Distributor.

  User can enter details such as Credit Limit, Credit Availability, Guarantee, Deposits and Trust if exists.

  Flags such as Cash sales flag, discount flag, Partial Dispatch Flag, Back orders Flag and Proforma Flag can be set here. User can also enter market share details of the Distributor.

 

After entering relevant details, user can click OK to save the Distributor or Cancel to discard the specifications and return to the 1st Screen.

 

Creating A NEW Distributor BASED ON AN EXISTING Distributor

To create a new Distributor, based on an existing Distributor, Select corresponding Distributor and click New Based On button.

 

The following interface is displayed where all details are copied from the Based on code except the Distributor Code which is a Unique Code for each Distributor. User is allowed free to edit necessary fields and save or discard his specifications.
