Route Maintenance

 

This Component is used to maintain details related to Routes in each territory.

At the initial load screen all available Routes will be listed in the grid.

 

1st Screen

 

v  By expanding the Selection Criteria, User will be able to filter Routes accordingly.

 

Selection could be based on PSA Code, Type, Category and Territory Code. Searching can be done on Start With, Any Where or End With options. User can also view the active and inactive records by unticking the Active check box.

 

v  There are different Options associated with the Main screen.

 

Add New Route to the System

  Click New Button

 

Add New Route Based on an existing Route

  Select specific Route New Based On Button

 

Edit specific Route in the System

  Select Route Edit Button

 

Exit from Route Maintenance

  Click Exit Button

 

 

v  In the Route Creation and Editing Process following fields are mandatory.

  Territory Code

  PSA Code

  PSA Description

  PSA Type

  PSA Category

  Default Executive

  Start Date

 

 

 

Creating a new Route

Creating A completely New Route

  User can create a completely new Route (PSA Code) by clicking the New button.

 

  When clicking the new button, following interface appears and the user will be able to enter the PSA attributes.

 

 

 

 

 

 

 

 

 

2nd Screen

 

  Territory Code: User should select the territory code for which the route is going to be created.

  PSA Code: A unique code which allows the user to identify the route (PSA) and differentiate from other routes.

  PSA Description: This is a mandatory field which allows the user to enter the description of the route.

 

 

 

v  PSA Header tab

  PSA Type: User should select the route operation type by using the prompt, from the predefined operation types. (Refer: Parameter, to find how to define operation types.)

  PSA Category: User should select the PSA category from the prompt.

  Monthly frequency: User can enter the number of times travel on the particular route per month.

  Distance (In KM.): Distance of the route in kilometers will be entered here.

  Duration (In Hrs.): The estimated time duration to cover the route will be entered here.

  Volume: The volume to be carried on the route will be entered here.

  Target Share (%): The user can enter target share value as a percentage, in this field.

  Default Executive: The user should select the default route executive using the prompt. These executives will load to the screen according to the PSA type which selected.

 

  Route Executives Panel:

  Executive Code: Allows adding the executive code through the prompt.

  Start Date: The start date of the route should be added here.

  End Date: Allows the user to add the end date to the system.

  Status: Determines whether the added route executive is active or inactive for that route.

After entering Details of Route Executives panel, the user can add the executive by tapping on the Add Button. User can also edit the record by selecting the added record and do changes and u0date the existing record as well.

 

PSA Text: Allows the user to add appropriate text for the route.

  Carry History: Carry History check box should be carried if balance quantity should be carried.

  Active: This flag determine whether the route is active or not. This flag will be automatically ticked when creating a new route.

 

v  PSA Details tab

 

Using this tab the user can attach outlets for the route. The available outlets for the territory will be loaded in the left side of the screen where the already selected outlets for the route will be loaded in the right side. Expanding the selection criteria panel will allow the user to filter outlets accordingly.

 

 

User can filter the outlets basedon Retailer classifications and Gegraphical classifications. Other than that filteration could be done on other route specifications as well. Search criterias could be Start with, Any where and Ends With.

 

After loading the otlets, user can tick on the outlets which are going to be attached to the route. By selecting and clicking on the exchange button (Green coloured arrow), user can attach the outlet to the route. User can also detach the outlets by selecting the perticular outlet from the selected list and clikcing on the exchange button towards the available outlets.

 

v  Transport Details tab

 

In this tab user can determine the Default Transportation and Route Transportation details by selecting the relevant information from the prompt.

 

After entering relevant details, user can click Ok to confirm the specifications or Cancel to discard the specifications.

 

 

Creating A NEW Route BASED ON AN EXISTING Route code

 

To create a new Route, based on an existing Route, Select corresponding Route and click New Based On button.

 

The following interface is displayed where all details are copied from the Based on code except the Route Code which is a Unique Code for each Route. User is allowed free to edit necessary fields and save or discard his specifications.

 

 

EDITING a Route

  User can edit a Route by selecting a Route from the grid and tapping on the Edit button.

 

  Following interface is displayed after selecting the edit option.

 

 

  User can edit the relevant fields and click OK or Cancel button to save or discard his specifications accordingly.
