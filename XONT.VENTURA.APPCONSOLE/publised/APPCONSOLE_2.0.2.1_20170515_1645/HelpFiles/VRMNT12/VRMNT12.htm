Retailer Maintenance

 

This Component is used to maintain details related to Retailers in each territory.

At the initial load screen all available Retailers will be listed in the grid.

 

1st Screen

 

 

 

 

 

 

 

 

 

v  By expanding the Selection Criteria, User will be able to filter Retailers accordingly.

Selection could be based on Customer Classifications or other retailer attributes. User can also view the active and inactive records by unticking the Active check box.

 

v  There are different Options associated with the Main screen.

 

Add New Retailer to the System

  Click New Button

 

Add New Retailer Based on an existing Retailer

  Select specific Retailer New Based On Button

 

Edit specific Retailer in the System

  Select Retailer Edit Button

 

Exit from Retailer Maintenance

  Click Exit Button

 

 

v  In the Retailer Creation and Editing Process following fields are mandatory.

  Retailer Code

  Retailer Name

  Address Line 1

  Post Code

  Alpha Search Code

  Short Address

  Trade Scheme Group

  Price Group

  Pricing Method

  Settlement Terms Code

  Payment Method Code

  All Geographical Classifications

2nd Screen

 

  Retailer Code: A unique code which allows the user to identify the retailer and differentiate from other retailers.

Retailer code can be created manually or automatically. Automatic Retailer Creation can be enabled by selecting the Auto retailer creation flag in RD Control maintenance (Refer: RD.Control Maintenance).

Mean time the Retailer Prefix, Retailer Code Length and Auto Retailer Fill Value can be determined in Territory Control (Refer: Territory Control Maintenance.)

Although these values are filled in territory control, they will be applied only when corresponding flags are enabled in RDControl maintenance (Refer: RD.Control Maintenance).

  Retailer Name: Allows the user to enter the retailer name.

 

 

v  Profile tab

  Full Address:

  Address Line 1/2/3/4/5: Allows the user to enter the retailers address. User can add the address in Address Line 1/2/3/4/5 fields.

  Post Code: Allows the user to add the post code of the retailer within this field.

 

  Details Tab:

Allows the user to add retailers details such as Alpha search code(), Short address(), Telephone Number, Email Address, Contact Name, Fax Number, Tel No Sys and web address of the retailer.

 

  Business Registration Detail:

The user can add the Business Registration details of the retailer to the system. Details such as Business registration number, type, Name, Date Expiry date and reference Code will be entered by using this option.

 

  Tax Details:

Retailer tax details will be maintained here. Drop down menu will allow selecting the Tax types which will be maintained in Tax maintenance (Refer: Tax Maintenance). After selecting the tax type, the user will be able to state whether the customer is Tax registered or not, by ticking on the Tax registered check box.

Status flag will allow enabling or disabling the Tax for the Retailer.

 

  Active:

The user will be able to determine whether the retailer is active or inactive by ticking or unticking the check box.

Similarly the user will also be able to select an active type of the retailer. (Normal, On stop, On Trial, Black Listed)

 

 

 

 

 

  Classification tab

 

  This tab allows the user to enter the classification details of the retailer. Each of these codes are selected from the predefined classification codes and values maintained in the classification code (Refer: Classification Code Maintenance).

 

  Similarly the user should select the Trade Scheme group, Price Group and pricing Method of the retailer.

 

  Retailer Executives:

In this option the user will be able to select the executives for the retailer. User will be facilitated with a drop down menu to select the executives with relevant operation types. The operation types are selected from the predefined parameters in parameter maintenance (Refer: Parameter Maintenance)

 

  Retailer Geographic Demarcation:

The user will be able to select the Geographical Demarcations using this panel. These geographical demarcations also will be maintained in Classification Code/ Classification Value. (Refer: Classification Code Maintenance)

v  Other Details tab

In this option will allow the user to enter details such as Settlement terms code (Refer: Settlement Term maintenance), Payment method code (Refer: Payment Method Maintenance) and other details such as credit limit and market share details.

 

 

 

 

v  Bank tab:

User can select the retailers bank details in this tab.

 

 

v  Contacts tab:

Retailer Contact details will be maintained in this tab. Details such as Telephone Number, Fax Number and email address will be maintained here. After entering relevent details, user can click the Add button in order to affix the information.

v  Delivary Address tab:

Retailer Delivery Details will be maintained in this tab. After entering relevent details, user can click the Add button in order to affix the information.

 

v  Invoice Address tab:

This tab will be used to maintain the invoice address details of the retailer. After entering relevent details, user can click the Add button in order to affix the information.

After entering relevant details, user can click OK to save the Retailer or Cancel to discard the specifications and return to the 1st Screen.

Creating A NEW Retailer BASED ON AN EXISTING Retailer

 

 

To create a new Retailer, based on an existing Retailer, Select corresponding Retailer and click New Based On button.

 

The following interface is displayed where all details are copied from the Based on code except the Retailer Code which is a Unique Code for each Retailer. User is allowed free to edit necessary fields and save or discard his specifications.
